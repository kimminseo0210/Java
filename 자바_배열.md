Java 배열에 관하여
==

### 1. 일반 배열`[]`(Array) : "예약제로 운영되는 독방"
- 특징 : 처음에 <b>"몇 칸짜리 만들래?"</b>하고 정하면 <b>절대 크기를 바꿀 수 없음</b>
- 장점 : 구조가 아주 단순해서 가볍고, <b>데이터를 읽고 쓰는 속도가 무지하게 빠름</b>, 메모리도 딱 필요한 만큼만 차지
- 단점 : 처음에 10칸을 만들었는데, 데이터가 11개가 들어오는 경우 바로 에러(`ArrayIndexOutOfBoundsException`)가 터짐, 반대로 100만 칸을 만들었는데 1개만 들어오면 그만큼 메모리 낭비
- 예시 : `int[] arr = new int[5]` 딱 5칸의 배열을 가짐


### 2. `ArrayList` : "무한 확장 가능한 고무줄 방"  
- 특징 : 크기를 미리 정할 필요가 없음. 데이터를 넣는 대로 <b>크기가 알아서 늘어남</b>
- 장점 : 데이터가 몇 개 들어올지 모를 때 무지하게 편함. 데이터를 다루기 위한 도구(메서드)들을 잔떡 가지고 있음
- 단점 : 스스로 방을 늘리고 줄이는 과정이 필요하기 때문에 일반 배열(`[]`)보다 조금 무겁고 미세하게 느림, `int`같은 기본 자료형을 직접 담지 못하고 `Integer`같은 객체 형태로 감싸서 넣어야함
- 예시 : `ArrayList<Integer> list = new ArrayList<>();` (끝없이 계속 들어감!)


| 구분       | `[]` (일반 배열)           | `ArrayList                         |
|----------|------------------------|------------------------------------|
| 크기(길이)   | 고정됨(변경 불가)             | 유동적(알아서 늘줄                         |
| 데이터 타입   | 기본형(`int,double` 등) 가능 | 객체만 가능(`Integer,Double` 등)         |
| 속도       | 아주 빠름                  | 상대적으로 미세하게 느림                      |
| 편의 기능    | 거의 없음                  | 아주 많음(`add,get,remove,contains` 등) |
| 길이 확인법   | `arr.lenght`           | `list.size()`                      |
| 데이터 꺼낼 때 | `arr[0]`               | `list.get(0)`                      |
